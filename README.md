# Sneaker Interaction

Сайт-витрина магазина оригинальных кроссовок, одежды и аксессуаров (Nike SB, Air Jordan,
KITH, Supreme и др.). Без корзины и онлайн-оплаты — заказ оформляется через Telegram.
Каталог редактируется через встроенную админ-панель прямо в браузере.

Статический фронтенд: чистые HTML, CSS и JavaScript, без сборки. Каталог и вход
модераторов работают на [Supabase](https://supabase.com) (опционально — см. ниже).

## Страницы

| Файл                 | Назначение                                                          |
| -------------------- | ------------------------------------------------------------------- |
| `index.html`         | Главная: hero, дропы, категории, лукбук, шаги заказа                 |
| `catalog.html`       | Каталог с фильтрами (категория, бренд, размер, наличие) и поиском    |
| `product.html`       | Карточка товара (`product.html?id=...`): галерея, размеры, заказ     |
| `admin.html`         | Админ-панель: вход модератора, добавление/редактирование товаров     |
| `catalog-print.html` | Версия каталога для печати / сохранения в PDF                       |
| `404.html`           | Страница «не найдено»                                                |

## Структура

```
.
├── index.html · catalog.html · product.html · admin.html · catalog-print.html · 404.html
├── robots.txt · sitemap.xml · supabase-setup.sql
└── assets/
    ├── css/   base.css (дизайн-система) + по странице: home/catalog/product/admin/print/notfound
    ├── js/    data.js (стартовый каталог) · config.js (ключи Supabase) · store.js (хранилище)
    │          · common.js (общее поведение) + по странице: home/catalog/product/admin/print/notfound
    ├── fonts/ Benzin (фирменный шрифт)
    └── img/   products/ · editorial/ · логотипы · favicon · og-cover
```

Дизайн-система (цвета, типографика, навигация, подвал, карточки, поиск, мобильное меню)
живёт в `assets/css/base.css` и `assets/js/common.js`. Каждая страница подключает базу
плюс свой небольшой файл стилей и логики.

## Два режима каталога

Хранилище (`assets/js/store.js`) автоматически выбирает режим:

- **Облако (Supabase)** — если в `assets/js/config.js` заданы реальные ключи. Каталог
  общий для всех, модератор входит по логину и правит товары прямо из браузера —
  изменения сразу видят все посетители.
- **Локальный фолбэк** — если ключей нет. Каталог берётся из `assets/js/data.js`,
  правки админки сохраняются в `localStorage` текущего браузера (видны только на этом
  устройстве). Удобно для разработки и предпросмотра.

`assets/js/data.js` — это «стартовый» каталог (54 позиции): он показывается, пока облако
не настроено или пусто, и используется кнопкой «Залить каталог в облако».

## Настройка Supabase (общий каталог + вход модераторов)

1. Зарегистрируйтесь на [supabase.com](https://supabase.com), создайте новый проект.
2. **SQL Editor → New query** → вставьте содержимое `supabase-setup.sql` → **Run**
   (создаст таблицу `products` и политики доступа: чтение всем, запись — вошедшим).
3. **Authentication → Users → Add user** → задайте e-mail и пароль модератора и
   включите **Auto Confirm User** (иначе вход не сработает). Так заводится каждый модератор.
4. **Project Settings → API** → скопируйте **Project URL** и **anon public** ключ,
   вставьте их в `assets/js/config.js`.
5. Откройте `admin.html`, войдите под модератором и нажмите **«Залить каталог в облако»**
   один раз — стартовые 54 товара уедут в базу.
6. Закоммитьте `config.js` и запушьте. `anon`-ключ безопасно держать в публичном коде —
   доступ ограничен политиками RLS.

> Дальнейшее редактирование: модератор открывает `admin.html`, входит и меняет товары —
> правки применяются ко всем мгновенно, git и перевыкладка не нужны.

## Локальный запуск

```bash
python3 -m http.server 8000
# затем откройте http://localhost:8000
```

(Через `file://` открывать не стоит — относительные пути, шрифты и Supabase работают по HTTP.)

## Публикация на GitHub Pages

1. Запушьте проект в репозиторий (репозиторий может быть приватным).
2. **Settings → Pages → Source: Deploy from a branch**, ветка `main`, папка `/ (root)`.
3. Через минуту сайт будет на `https://<логин>.github.io/<репозиторий>/`.
4. Подставьте этот адрес вместо `REPLACE-WITH-YOUR-DOMAIN` в `robots.txt` и `sitemap.xml`.

Сайт также без изменений разворачивается на Netlify или Vercel (без шага сборки).

## Что заменить на своё

- Telegram, e-mail, Instagram — в `assets/js/common.js` (`TG`) и в подвалах страниц
  (плейсхолдеры `t.me/sneaker_interaction`, `hello@sneaker-in.ru`).
- Домен в `robots.txt` и `sitemap.xml`.
- Ключи Supabase в `assets/js/config.js` (если нужен общий каталог).
